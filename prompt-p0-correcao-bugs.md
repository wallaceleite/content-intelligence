### BUG 1: Calculo de Engagement Rate Inflado (metrics.ts)

**Arquivo:** `src/lib/metrics.ts`, funcao `calculateDerivedMetrics`, linha 33

**Problema atual:**
```typescript
const views = post.views || 1; // fallback pra 1 quando views = 0
const engagementRate = (totalEngagement / views) * 100;
```

Quando um post tem `views = 0` (carrosseis antigos, imagens), o fallback `|| 1` faz o calculo explodir. Um post com 746 likes e 0 views mostra 74600% de engajamento.

**Correcao necessaria:**
- Se `views > 0`, usar views como denominador (comportamento atual)
- Se `views = 0` MAS tem likes > 0, usar `followers_count` do perfil como denominador (engajamento sobre base de seguidores). Como a funcao nao recebe followers hoje, usar `likes + comments` como base (engagement = totalEngagement / totalEngagement * 100 = 100% cap)
- Abordagem mais simples: se views = 0, usar `likes_count` como proxy de alcance minimo (quem curtiu pelo menos viu)
- O engagement nunca deve ultrapassar 100% — adicionar cap

**Implementacao sugerida:**
```typescript
// Use views if available; otherwise use likes as minimum reach proxy
const baseMetric = post.views > 0 ? post.views : (post.likes > 0 ? post.likes : 1);
const engagementRate = Math.min((totalEngagement / baseMetric) * 100, 100);
```

Aplique a mesma logica no `filterAndRankPosts` (linhas 78-82 e 85-88) que tambem tem o mesmo problema com `views || 1`.

Alem disso, aplique o cap tambem no `salesPotentialScore` (linha 51-54) que usa `Math.max(views, 1)`.
